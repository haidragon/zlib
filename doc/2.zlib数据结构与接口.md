# zlib数据结构与接口分析

## 一、核心数据结构

### 1. z_stream - 流处理结构

**位置**: `zlib.h:87-106`

```c
typedef struct z_stream_s {
    z_const unsigned char *next_in;     /* 下一个输入字节指针 */
    unsigned int     avail_in;          /* next_in处可用的字节数 */
    unsigned long    total_in;          /* 到目前为止读取的输入字节总数 */

    unsigned char    *next_out;         /* 下一个输出字节应放置的位置 */
    unsigned int     avail_out;         /* next_out处剩余的可用空间 */
    unsigned long    total_out;         /* 到目前为止输出的字节总数 */

    z_const char *msg;                  /* 最后的错误消息，无错误时为NULL */
    struct internal_state *state;       /* 内部状态，应用程序不可见 */

    alloc_func zalloc;                  /* 用于分配内部状态的函数 */
    free_func  zfree;                   /* 用于释放内部状态的函数 */
    void      *opaque;                   /* 传递给zalloc和zfree的私有数据对象 */

    int     data_type;                  /* 数据类型的最佳猜测：二进制或文本 */
    unsigned long   adler;               /* 未压缩数据的adler32校验值 */
    unsigned long   reserved;           /* 保留供将来使用 */
} z_stream;

typedef z_stream *z_streamp;
```

**作用**: 压缩/解压流的核心数据结构，管理输入输出缓冲区和内部状态。

**关键字段说明**:
- `next_in/next_out`: 输入/输出缓冲区指针
- `avail_in/avail_out`: 可用的输入/输出字节数
- `total_in/total_out`: 处理的总字节数，用于统计和进度报告
- `state`: 内部实现状态（压缩时为`internal_state`，解压时为`inflate_state`）
- `zalloc/zfree`: 自定义内存管理函数
- `opaque`: 传递给内存管理函数的私有数据

**内存管理函数类型**:
```c
typedef void *(*alloc_func)(void *opaque, unsigned int items, unsigned int size);
typedef void   (*free_func)(void *opaque, void *address);
```

### 2. gz_header - gzip文件头结构

**位置**: `zlib.h:114-131`

```c
typedef struct gz_header_s {
    int           text;       /* 如果压缩数据被认为是文本则为true */
    unsigned long time;       /* 修改时间 */

    int           os;         /* 操作系统 */

    unsigned int  extra_max;  /* extra处的空间（仅当读取头部时） */
    unsigned char *name;      /* 指向以零结尾的文件名或NULL */
    unsigned int  name_max;   /* name处的空间（仅当读取头部时） */
    unsigned char *comment;   /* 指向以零结尾的注释或NULL */
    unsigned int  comm_max;   /* comment处的空间（仅当读取头部时） */

typedef gz_header *gz_headerp;
```

**作用**: 存储gzip文件的头部信息，符合RFC 1952规范。

### 3. internal_state - 压缩内部状态结构

**位置**: `deflate.h:88-249`

```c
typedef struct internal_state {
    z_stream *strm;              /* 指向zlib流的指针 */
    int   status;                /* 状态 */
    unsigned char *pending_buf;  /* 仍需输出的数据 */
    unsigned long  pending_buf_size; /* pending_buf的大小 */
    unsigned char *pending_out;  /* 下一个输出到流的待定字节 */
    unsigned int   pending;      /* 待定缓冲区中的字节数 */
    int   wrap;                  /* bit 0为true表示zlib，bit 1为true表示gzip */
    gz_header     *gzhead;       /* 要写入的gzip头部信息 */
    unsigned int   gzindex;      /* extra、name或comment中的位置 */
    unsigned char  method;       /* 只能是DEFLATED */
    int   last_flush;            /* 上一次deflate调用的flush参数值 */

    /* deflate.c使用的字段 */
    unsigned int w_size;         /* LZ77窗口大小（默认32K） */
    unsigned int w_bits;         /* log2(w_size) (8..16) */
    unsigned int w_mask;         /* w_size - 1 */

    unsigned char *window;       /* 滑动窗口 */
    unsigned long window_size;   /* 窗口的实际大小：2*wSize */

    Pos *prev;                   /* 具有相同哈希索引的较旧字符串的链接 */
    Pos *head;                   /* 哈希链的头节点或NIL */

    unsigned int ins_h;          /* 要插入的字符串的哈希索引 */
    unsigned int hash_size;      /* 哈希表中的元素数量 */
    unsigned int hash_bits;      /* log2(hash_size) */
    unsigned int hash_mask;      /* hash_size-1 */
    unsigned int hash_shift;     /* 每次输入步骤ins_h必须移位的位数 */

    long block_start;            /* 当前输出块开始时的窗口位置 */

    unsigned int match_length;   /* 最佳匹配的长度 */
    IPos prev_match;             /* 上一个匹配 */
    int match_available;         /* 如果存在上一个匹配则设置 */
    unsigned int strstart;       /* 要插入的字符串的起始位置 */

    unsigned int match_start;    /* 当前匹配的起始位置 */
    unsigned int lookahead;      /* 当前窗口中可用的字节数 */

    unsigned int prev_length;   /* 上一个匹配的长度 */
    unsigned int max_chain_length; /* 哈希链中的最大搜索长度 */
    unsigned int max_lazy_match; /* 延迟匹配的最大长度 */

    int level;                   /* 压缩级别 */
    int strategy;                /* 压缩策略 */

    unsigned int good_match;     /* 用于懒惰匹配的良好匹配值 */
    int nice_match;              /* 停止搜索的好匹配值 */

    /* 滑动窗口的树结构 */
    ct_data *dyn_ltree;          /* 动态字面值/长度树 */
    ct_data *dyn_dtree;          /* 动态距离树 */
    ct_data *bl_tree;            /* 霍夫曼树的位长度树 */
    
    struct tree_desc_s l_desc;   /* 字面值树描述符 */
    struct tree_desc_s d_desc;   /* 距离树描述符 */
    struct tree_desc_s bl_desc;  /* 位长度树描述符 */

    unsigned int bl_count[BL_CODES]; /* 位长度计数 */
    int heap[2*L_CODES+1];       /* 用于建立树的堆 */
    int heap_len;                /* 堆中活跃元素的数量 */
    int heap_max;                /* 堆的最大元素 */
    
    unsigned char depth[2*L_CODES+1]; /* 每个节点的深度 */
    
    unsigned int lit_bufsize;    /* 字面值缓冲区大小 */
    unsigned long l_buf;         /* 字面值和长度缓冲区的索引 */
    
    unsigned int last_lit;       /* 上一个字面值 */
    unsigned int d_buf;          /* 距离缓冲区索引 */
    
    unsigned int opt_len;        /* 优化代码长度 */
    unsigned int static_len;     /* 静态代码长度 */
    unsigned int matches;        /* 匹配的字节数 */
    unsigned int insert;         /* 插入的字节数 */
    
    unsigned int compressed_len; /* 压缩长度 */
    unsigned long bits_sent;     /* 发送的总位数 */
} internal_state;
```

**作用**: 压缩过程中的内部状态管理，包含LZ77滑动窗口、哈希表、霍夫曼树等。

## 四、数据流转示意图

### 压缩流程
```
输入数据 → z_stream.next_in → deflate() → z_stream.next_out → 输出数据
         ←─── avail_in ───→              ←─── avail_out ───→
```

### 解压流程
```
压缩数据 → z_stream.next_in → inflate() → z_stream.next_out → 原始数据
         ←─── avail_in ───→              ←─── avail_out ───→
```

## 五、内存布局

### deflate内部状态内存占用
- 滑动窗口：32KB × 2 = 64KB
- 哈希表：根据windowBits动态分配
- 霍夫曼树：约1-2KB
- 总计：约70-100KB

### inflate内部状态内存占用
- 滑动窗口：根据windowBits（默认32KB）
- 解码表：约10KB
- 总计：约50-60KB

## 六、典型使用模式

### 模式1：一次性压缩
```c
Bytef *dest = malloc(compressBound(sourceLen));
uLongf destLen = compressBound(sourceLen);
compress(dest, &destLen, source, sourceLen);
```

### 模式2：流式压缩
```c
z_stream strm;
deflateInit(&strm, Z_DEFAULT_COMPRESSION);
// 循环处理数据
strm.next_in = input;
strm.avail_in = input_len;
strm.next_out = output;
strm.avail_out = output_len;
deflate(&strm, Z_FINISH);
deflateEnd(&strm);
```

### 模式3：文件压缩
```c
gzFile gf = gzopen("output.gz", "wb");
gzwrite(gf, buffer, len);
gzclose(gf);
```

## 七、注意事项

1. **内存管理**: 使用自定义内存管理函数时，确保线程安全
2. **错误处理**: 始终检查返回值，处理错误情况
3. **缓冲区大小**: 使用`compressBound()`计算所需缓冲区大小
4. **流状态**: 同一z_stream结构不能同时用于压缩和解压
5. **版本兼容**: 使用`zlibVersion()`检查版本兼容性
6. **多线程**: 每个线程使用独立的z_stream结构
