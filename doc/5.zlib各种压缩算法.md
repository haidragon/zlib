# zlib各种压缩算法

## 一、zlib压缩算法概览

zlib提供了多种压缩策略和算法，根据不同的压缩级别和数据类型选择最适合的方法。

```
zlib压缩算法体系
├── 压缩策略
│   ├── Z_DEFAULT_STRATEGY (默认策略)
│   ├── Z_FILTERED (过滤数据策略)
│   ├── Z_HUFFMAN_ONLY (仅霍夫曼编码)
│   ├── Z_RLE (行程编码)
│   └── Z_FIXED (固定霍夫曼编码)
│
├── 压缩模式
│   ├── deflate_stored (存储模式)
│   ├── deflate_fast (快速压缩)
│   ├── deflate_slow (慢速但高压缩率)
│   ├── deflate_rle (RLE压缩)
│   └── deflate_huff (仅霍夫曼压缩)
│
└── 压缩级别
    ├── 0: 无压缩 (deflate_stored)
    ├── 1-3: 快速压缩 (deflate_fast)
    ├── 4-9: 平衡到最高压缩率 (deflate_slow)
    └── -1: 默认级别 (6)
```

## 二、压缩策略 (Compression Strategy)

### 2.1 策略类型定义

```c
// 在zlib.h中定义
#define Z_FILTERED            1       /* 过滤数据策略 */
#define Z_HUFFMAN_ONLY        2       /* 仅霍夫曼编码 */
#define Z_RLE                 3       /* 行程编码 */
#define Z_FIXED               4       /* 固定霍夫曼编码 */
#define Z_DEFAULT_STRATEGY    0       /* 默认策略 */
```

### 2.2 Z_DEFAULT_STRATEGY (默认策略)

**特点**:
- 综合考虑压缩率和速度
- 适合大多数类型的数据
- 自动选择最佳的压缩方法

**适用场景**:
- 通用数据压缩
- 不确定数据类型时
- 需要平衡性能的场景

**工作原理**:
```c
// 在deflate.c中配置
switch (s->strategy) {
    case Z_DEFAULT_STRATEGY:
        // 使用懒惰匹配和最长匹配
        // 根据压缩级别调整参数
        break;
}
```

### 2.3 Z_FILTERED (过滤数据策略)

**特点**:
- 针对高度结构化的数据优化
- 假设数据已经被某种方式过滤或处理
- 避免长距离匹配

**适用场景**:
- 已经预过滤的数据
- 具有局部相关性的数据
- 需要快速处理结构化数据

**实现原理**:
```c
// 在deflate.c中
case Z_FILTERED:
    // 调整匹配参数
    // 减少max_lazy_match
    // 限制匹配长度和距离
    s->max_lazy_match = configuration_table[level].max_lazy;
    break;
```

**配置参数**:
```c
// 配置表中的过滤策略参数
static const config configuration_table[10] = {
    /* good lazy nice chain */
    /* 4 */ {4,    4, 16,   16, deflate_slow},  /* 懒惰匹配 */
    /* 5 */ {8,   16, 32,   32, deflate_slow},
    /* 6 */ {8,   16, 128, 128, deflate_slow},
    ...
};
```

### 2.4 Z_HUFFMAN_ONLY (仅霍夫曼编码)

**特点**:
- 禁用LZ77匹配
- 仅使用霍夫曼编码压缩
- 速度快但压缩率较低
- 适合LZ77效率低的数据

**适用场景**:
- 随机数据或已压缩数据
- 小数据块
- LZ77查找开销大于收益的情况

**源码实现**:
```c
// 在deflate.c中
static block_state deflate_huff(deflate_state *s, int flush) {
    /* 完全跳过LZ77匹配 */
    /* 只进行霍夫曼编码 */
    
    for (;;) {
        /* 直接输出字面值 */
        if (s->strstart < s->block_start) {
            /* 处理剩余的字面值 */
        }
        
        /* 发送块结束 */
        if (flush == Z_FINISH) {
            return finish_done;
        }
    }
}
```

**压缩效果对比**:
```
原始数据: 1000字节
- Z_DEFAULT_STRATEGY: 压缩后 ~250字节 (4:1)
- Z_HUFFMAN_ONLY:  压缩后 ~600字节 (1.67:1)
- 速度提升: 约30-50%
```

### 2.5 Z_RLE (行程编码策略)

**特点**:
- 结合RLE和霍夫曼编码
- 适合有大量连续重复值的数据
- 比纯LZ77更快

**适用场景**:
- 图像数据（特别是像素值）
- 游戏地图数据
- 具有大量重复模式的数据

**源码实现**:
```c
// 在deflate.c中
static block_state deflate_rle(deflate_state *s, int flush) {
    unsigned int prev_length = s->prev_length;
    unsigned int strstart = s->strstart;
    unsigned int match_length = 0;
    
    /* 检查连续重复字符 */
    if (s->strstart > 0 && 
        s->window[s->strstart - 1] == s->window[s->strstart]) {
        match_length = 1;
        /* 统计连续重复次数 */
        while (match_length < MAX_MATCH &&
               s->strstart + match_length < s->window_size &&
               s->window[s->strstart - 1] == 
               s->window[s->strstart + match_length]) {
            match_length++;
        }
    }
    
    /* 如果找到足够长的RLE序列 */
    if (match_length >= MIN_MATCH) {
        /* 输出距离为1的匹配 */
        /* 距离1表示重复上一个字符 */
    }
}
```

**压缩示例**:
```
原始: AAAAAAAAABBBBBCCCCC
RLE:  (1,9)A, (1,5)B, (1,5)C
    距离  长度
```

### 2.6 Z_FIXED (固定霍夫曼编码)

**特点**:
- 使用预定义的固定霍夫曼树
- 不需要传输霍夫曼树
- 速度快，压缩率略低

**适用场景**:
- 需要快速编码/解码
- 数据块较小
- 传输霍夫曼树开销较大时

**固定霍夫曼树**:
```c
// 在trees.h中定义
static const ct_data static_ltree[L_CODES+2] = {
    /* 长度/字面值固定树 */
    /* 0-143: 8位编码 */
    /* 144-255: 9位编码 */
    /* 256-279: 7位编码 */
    /* 280-287: 8位编码 */
};

static const ct_data static_dtree[D_CODES] = {
    /* 距离固定树 */
    /* 0-31: 5位编码 */
};
```

**优势对比**:
```
动态霍夫曼:
- 更高的压缩率
- 需要传输霍夫曼树 (约100-300字节)
- 编码速度略慢

固定霍夫曼:
- 编码速度快
- 无需传输霍夫曼树
- 压缩率略低 (约5-10%)
```

## 三、压缩模式 (Compression Mode)

### 3.1 deflate_stored (存储模式)

**特点**:
- 不进行任何压缩
- 原始数据直接存储
- 用于数据本身已经压缩或随机数据

**配置**:
```c
// 在deflate.c配置表中
static const config configuration_table[10] = {
    /* 0 */ {0, 0, 0, 0, deflate_stored},  /* 存储模式 */
    ...
};
```

**源码实现**:
```c
static block_state deflate_stored(deflate_state *s, int flush) {
    unsigned long max_blks = 0xffff;
    unsigned long min_block = MIN_BLOCK;
    unsigned len, stored_len, block_len;
    
    /* 存储块的头部 */
    /* 3位块类型 + 5位补零 = 1字节 */
    /* 16位长度 + 16位补码 = 4字节 */
    /* 总共5字节头部开销 */
    
    if (s->block_start >= 0) {
        /* 计算当前块大小 */
        len = s->strstart - s->block_start;
    }
    
    /* 如果块太小或随机数据，使用存储模式 */
    if (len <= 0 || len < s->block_size) {
        /* 继续收集数据 */
        return need_more;
    }
    
    /* 存储数据块 */
    put_short(s, len);      /* 长度 */
    put_short(s, ~len);     /* 长度的补码 */
    
    /* 直接复制数据 */
    memcpy(s->pending_buf + s->pending, 
           s->window + s->block_start, 
           len);
    
    return block_done;
}
```

**适用条件**:
- 压缩级别为0 (Z_NO_COMPRESSION)
- 数据本身已经是压缩格式
- 数据随机性高，压缩效果差
- 块大小小于最小压缩块

**数据格式**:
```
┌─────────┬──────────┬────────────┬──────────┐
│ BFINAL  │ BTYPE    │ LEN (16位) │ NLEN(16位)│
│ (1位)   │ (2位)    │            │          │
├─────────┼──────────┼────────────┼──────────┤
│ 最后块? │ 00:存储  │ 块长度     │ 长度补码 │
└─────────┴──────────┴────────────┴──────────┘
```

### 3.2 deflate_fast (快速压缩)

**特点**:
- 针对速度优化
- 压缩级别1-3使用
- 快速查找，减少匹配开销
- 适合实时压缩场景

**配置参数**:
```c
// 在deflate.c中
static const config configuration_table[10] = {
    /* good lazy nice chain */
    /* 1 */ {4,    4,  8,    4, deflate_fast}, /* 最大速度 */
    /* 2 */ {4,    5, 16,    8, deflate_fast},
    /* 3 */ {4,    6, 32,   32, deflate_fast},
    ...
};
```

**参数说明**:
- `good_length`: 好匹配长度阈值
- `max_lazy`: 最大懒惰匹配长度
- `nice_length`: 优秀匹配长度阈值
- `max_chain`: 最大哈希链搜索长度

**源码实现**:
```c
static block_state deflate_fast(deflate_state *s, int flush) {
    IPos hash_head;          /* 哈希链头 */
    int bflush;              /* 块刷新状态 */
    
    /* 快速模式不使用懒惰匹配 */
    /* 每次找到一个匹配就立即使用 */
    
    for (;;) {
        /* 插入当前字符串到哈希表 */
        INSERT_STRING(s, s->strstart, hash_head);
        
        /* 查找匹配 */
        if (hash_head != NIL && 
            s->strstart - hash_head <= MAX_DIST(s)) {
            /* 找到潜在匹配 */
            s->match_length = longest_match(s, hash_head);
            
            /* 快速模式：立即使用匹配 */
            if (s->match_length > MIN_MATCH) {
                /* 输出匹配引用 */
                bflush = tr_tally_dist(s, 
                    s->strstart - s->match_start, 
                    s->match_length - MIN_MATCH);
                
                /* 前进到匹配之后 */
                s->lookahead -= s->match_length;
                s->strstart += s->match_length;
                s->match_length = 0;
            } else {
                /* 输出字面值 */
                bflush = tr_tally_lit(s, s->window[s->strstart]);
                s->lookahead--;
                s->strstart++;
            }
        } else {
            /* 无匹配，输出字面值 */
            bflush = tr_tally_lit(s, s->window[s->strstart]);
            s->lookahead--;
            s->strstart++;
        }
        
        /* 填充窗口 */
        if (s->lookahead < MIN_LOOKAHEAD) {
            fill_window(s);
        }
        
        /* 检查是否完成 */
        if (bflush) FLUSH_BLOCK(s, 0);
    }
}
```

**性能特点**:
```
压缩速度:  ★★★★★
压缩率:    ★★☆☆☆
内存使用:  ★★★☆☆
适用:      实时压缩、小数据块
```

### 3.3 deflate_slow (慢速但高压缩率)

**特点**:
- 针对压缩率优化
- 压缩级别4-9使用
- 使用懒惰匹配机制
- 深度搜索匹配

**配置参数**:
```c
// 在deflate.c中
static const config configuration_table[10] = {
    /* good lazy nice chain */
    /* 4 */ {4,    4, 16,   16, deflate_slow},
    /* 5 */ {8,   16, 32,   32, deflate_slow},
    /* 6 */ {8,   16, 128, 128, deflate_slow},
    /* 7 */ {8,   32, 128, 256, deflate_slow},
    /* 8 */ {32, 128, 258, 1024, deflate_slow},
    /* 9 */ {32, 258, 258, 4096, deflate_slow} /* 最大压缩 */
};
```

**懒惰匹配机制**:
```
懒惰匹配原理:
1. 找到一个匹配后不立即使用
2. 检查下一个位置的匹配
3. 如果下一个位置匹配更长，选择后者
4. 否则使用当前匹配

示例:
位置:  0123456789
数据:  ABRACADABRA

位置3的匹配: "AB" (距离2, 长度2)
位置4的匹配: "AD" (无匹配)

结果: 使用位置3的匹配
```

**源码实现**:
```c
static block_state deflate_slow(deflate_state *s, int flush) {
    IPos hash_head;
    int bflush;
    
    for (;;) {
        /* 插入当前字符串 */
        INSERT_STRING(s, s->strstart, hash_head);
        
        /* 查找匹配 */
        if (hash_head != NIL && 
            s->strstart - hash_head <= MAX_DIST(s)) {
            s->match_length = longest_match(s, hash_head);
        }
        
        /* 懒惰匹配逻辑 */
        if (s->match_length >= MIN_MATCH) {
            /* 检查是否应该延迟 */
            if (s->match_length <= s->max_lazy_match) {
                /* 保存匹配，继续向前 */
                s->match_available = 1;
                s->strstart++;
                s->lookahead--;
            } else {
                /* 匹配足够长，立即使用 */
                if (s->match_available) {
                    /* 先输出之前保存的匹配 */
                    tr_tally_lit(s, s->window[s->strstart - 1]);
                }
                /* 输出当前匹配 */
                bflush = tr_tally_dist(s,
                    s->strstart - s->match_start,
                    s->match_length - MIN_MATCH);
                
                s->lookahead -= s->match_length;
                s->strstart += s->match_length;
                s->match_length = 0;
            }
        } else {
            /* 无匹配 */
            if (s->match_available) {
                /* 输出保存的字面值 */
                bflush = tr_tally_lit(s, s->window[s->strstart - 1]);
                s->match_available = 0;
            }
            s->strstart++;
            s->lookahead--;
        }
        
        /* 填充窗口 */
        if (s->lookahead < MIN_LOOKAHEAD) {
            fill_window(s);
        }
        
        /* 刷新块 */
        if (bflush) FLUSH_BLOCK(s, 0);
    }
}
```

**性能特点**:
```
压缩速度:  ★★☆☆☆
压缩率:    ★★★★★
内存使用:  ★★★★☆
适用:      归档存储、离线压缩
```

### 3.4 deflate_rle (RLE压缩)

**特点**:
- 专门针对连续重复数据
- 比标准LZ77更高效
- 结合霍夫曼编码

**源码实现**:
```c
static block_state deflate_rle(deflate_state *s, int flush) {
    int bflush;
    unsigned int prev, cur, match;
    
    for (;;) {
        /* 检查RLE模式 */
        if (s->match_length >= MIN_MATCH) {
            /* 输出RLE序列 */
            bflush = tr_tally_dist(s, 1, s->match_length - MIN_MATCH);
            
            s->lookahead -= s->match_length;
            s->strstart += s->match_length;
            s->match_length = 0;
        } else {
            /* 检查连续重复 */
            if (s->strstart > 0) {
                prev = s->window[s->strstart - 1];
                cur = s->window[s->strstart];
                
                if (prev == cur) {
                    match = 2;
                    /* 统计重复次数 */
                    while (s->strstart + match < s->window_size &&
                           s->window[s->strstart - 1] == 
                           s->window[s->strstart + match] &&
                           match < MAX_MATCH) {
                        match++;
                    }
                    
                    if (match >= MIN_MATCH) {
                        s->match_length = match;
                        s->match_start = s->strstart - 1;
                    }
                }
            }
            
            if (s->match_length < MIN_MATCH) {
                /* 输出字面值 */
                bflush = tr_tally_lit(s, s->window[s->strstart]);
                s->lookahead--;
                s->strstart++;
            }
        }
        
        if (s->lookahead < MIN_LOOKAHEAD) {
            fill_window(s);
        }
        
        if (bflush) FLUSH_BLOCK(s, 0);
    }
}
```

**压缩示例**:
```
原始: AAAAAAAAAAABBBBBCCCC
标准LZ77: (10, A), (5, B), (4, C)
RLE:     (1,10)A, (1,5)B, (1,4)C

RLE优势:
- 距离编码更紧凑
- 适合大量连续重复
- 编码/解码简单
```

### 3.5 deflate_huff (仅霍夫曼编码)

**特点**:
- 完全跳过LZ77
- 只进行霍夫曼编码
- 适合随机数据

**源码实现**:
```c
static block_state deflate_huff(deflate_state *s, int flush) {
    int bflush;
    
    for (;;) {
        /* 检查是否所有数据已处理 */
        if (s->strstart >= s->block_start) {
            /* 准备发送块 */
            FLUSH_BLOCK(s, flush == Z_FINISH);
        }
        
        /* 直接输出字面值 */
        if (s->strstart < s->window_size) {
            bflush = tr_tally_lit(s, s->window[s->strstart]);
            s->strstart++;
        } else {
            /* 需要更多输入 */
            if (flush == Z_FINISH) {
                return finish_done;
            }
            return need_more;
        }
        
        /* 填充窗口 */
        if (s->lookahead < MIN_LOOKAHEAD) {
            fill_window(s);
        }
    }
}
```

## 四、压缩级别详解

### 4.1 级别0: Z_NO_COMPRESSION

```
配置: {0, 0, 0, 0, deflate_stored}
算法: 存储模式
速度: 最快
压缩率: 无
用途: 已压缩数据、随机数据
```

### 4.2 级别1-3: 快速压缩

```
级别1: {4, 4, 8, 4, deflate_fast}
级别2: {4, 5, 16, 8, deflate_fast}
级别3: {4, 6, 32, 32, deflate_fast}

特点:
- 最快速度
- 轻度压缩
- 实时应用
```

### 4.3 级别4-6: 平衡模式

```
级别4: {4, 4, 16, 16, deflate_slow}
级别5: {8, 16, 32, 32, deflate_slow}
级别6: {8, 16, 128, 128, deflate_slow}  // 默认

特点:
- 速度与压缩率平衡
- 懒惰匹配
- 通用场景
```

### 4.4 级别7-9: 最大压缩

```
级别7: {8, 32, 128, 256, deflate_slow}
级别8: {32, 128, 258, 1024, deflate_slow}
级别9: {32, 258, 258, 4096, deflate_slow}

特点:
- 最高压缩率
- 深度搜索
- 离线归档
```

## 五、算法选择决策树

```
开始压缩
   ↓
数据是否已压缩?
   ├─ 是 → Z_HUFFMAN_ONLY 或 Z_FIXED
   └─ 否 → 继续判断
        ↓
数据类型?
   ├─ 随机/高熵 → Z_HUFFMAN_ONLY
   ├─ 高度重复 → Z_RLE
   ├─ 结构化 → Z_FILTERED
   └─ 通用 → Z_DEFAULT_STRATEGY
        ↓
压缩需求?
   ├─ 速度优先 → 级别1-3 (deflate_fast)
   ├─ 平衡 → 级别4-6 (deflate_slow)
   └─ 压缩率优先 → 级别7-9 (deflate_slow)
```

## 六、实际应用建议

## 八、总结

### 关键要点

1. **策略选择**:
   - 通用数据: Z_DEFAULT_STRATEGY
   - 结构化数据: Z_FILTERED
   - 随机数据: Z_HUFFMAN_ONLY
   - 重复数据: Z_RLE

2. **级别选择**:
   - 实时应用: 1-3
   - 通用场景: 4-6 (默认6)
   - 离线归档: 7-9

3. **算法选择**:
   - 速度优先: deflate_fast
   - 压缩率优先: deflate_slow
   - 特殊数据: deflate_rle/deflate_huff

### 优化建议

1. 根据数据类型选择合适策略
2. 根据应用场景选择压缩级别
3. 测试不同配置找到最优参数
4. 考虑预处理以提升压缩效果
